# 출판 품질 번역 가이드라인
## "Lost and Founder" 한국어 번역

**목표**: 출판사 제출 수준의 전문적 일관성

---

## 📋 기본 원칙

### 1. **톤 & 스타일**

**핵심**: 비즈니스 교양서 / 전문적이면서 접근 가능한 / 성인 독자층

```
✅ 적절한 톤:
- 존댓말 통일: "~습니다", "~합니다"
- 문체: 명확하고 직설적이면서도 친근
- 어조: 저자의 개인적 이야기 → 따뜻하고 솔직함
- 리스트/설명: 객관적이고 명확함

❌ 피할 톤:
- 반말 혼재 ("~했다", "~하는데")
- 과도한 경어 (어색함)
- 기술 용어만 사용 (접근성 저하)
- 너무 친근한 문어 체 (비전문적)
```

### 2. **문장 길이 & 구조**

```
권장:
- 평균 문장 길이: 20-30단어
- 각 문단: 3-5문장
- 전환문: 명확한 논리적 연결

피하기:
- 40단어 이상 장문 (가독성 저하)
- 중첩된 문장 (번역 어려움)
- 애매한 대명사 참조
```

### 3. **용어 일관성**

**핵심 용어 사전** (이 가이드라인 하단 참고)

## 📝 번역 체크리스트 (각 섹션별)

각 섹션 번역 후 이 체크리스트로 검증:

```
□ 톤: 모든 문장이 존댓말("~습니다")로 통일됨
□ 용어: 위 사전의 용어가 일관성 있게 사용됨
□ 문장: 평균 20-30단어, 읽기 쉬운 길이
□ 논리: 문단 간 전환이 자연스러움
□ 이름: 고유명사(사람/회사명) 유지 또는 일관성 있게 번역
□ 기술 용어: 원문과 일치 (불필요한 번역 피함)
□ 가독성: 어색한 표현 없음
```

## 🎯 부분별 톤 가이드

### A. 개인 일화 섹션 (저자의 개인 경험)

```
톤: 진솔하고 따뜻하면서도 비판적 성찰
예시:
"나는 25세였고, 그의 도착에 완전히 당황했습니다.
하지만 저는 형편없는 거짓말쟁이라서 이렇게 말했습니다:
'죄송하지만 그 사람은 여기 없는 것 같습니다.'"

❌ 피할 것:
"나는 25살이었고, 그 사람이 왔는데 정말 깜짝 놀랐다."
```

### B. 통계/데이터 섹션

```
톤: 객관적이고 명확함, 증거 제시
예시:
"하버드 비즈니스 스쿨의 코호트 분석에 따르면,
초기 단계의 기술 회사 75% 이상이
투자자의 자본을 회수하지 못하고 있습니다."

✅ 명확하고 직설적
```

### C. 조언/교훈 섹션

```
톤: 직설적이면서 실용적, 강한 주장
예시:
"투명성은 처음에는 더 어렵습니다.
하지만 이것은 엄청나게 강력하며,
당신 주변의 모든 사람에게 긍정적인 영향을 미칩니다."

✅ 강하지만 지나치지 않음
```

## 🔄 섹션 간 연결 가이드

### 인접 섹션 연결

**문제**: 현재 각 섹션이 독립적 → 연결 약함

**해결**: 이전 섹션의 주제를 다음 섹션에서 언급

```
예시:
Section 2 끝: "...이 기업가들로부터 배운 것은?"
Section 3 시작: "그들이 내게 가르친 것은 정확히 이것입니다."
→ 자연스러운 흐름

현재 상태:
Section 2 끝: "...스타트업 치트 코드였습니다."
Section 3 시작: "# 한국어 번역" (완전히 끊김)
```

## ✅ 최종 검증 기준

**이 기준을 충족하면 출판 수준:**

1. **톤/스타일**: 100% 일관성 (존댓말, 전문적)
2. **용어**: 30개 핵심 용어 + 추가 용어 사전 일관성
3. **가독성**: 모든 문장이 한 번에 이해 가능
4. **논리 흐름**: 섹션 간 자연스러운 전환
5. **품질**: 어색한 표현 0개
6. **전문성**: 기술 용어 정확, 비즈니스 개념 정확
