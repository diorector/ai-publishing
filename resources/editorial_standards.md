# 출판 편집 기준 - Editorial Standards

## 출판물 편집 5대 원칙

### 1. 명확성 (Clarity)
**원칙**: 독자가 단 한 번 읽고도 정확히 이해할 수 있어야 한다.

**실천 방법**:
- 모호한 표현 제거: "~것 같다", "어느 정도", "대략" 등 애매한 표현 구체화
- 지시어 명확화: "그것", "이러한" 대신 구체적 명사 사용
- 주어와 술어 명확: 문장의 주체가 누구인지 분명히
- 한 문장, 한 생각: 여러 아이디어를 한 문장에 담지 않기

### 2. 간결성 (Conciseness)
**원칙**: 의미를 훼손하지 않으면서 최대한 짧고 명료하게.

**불필요한 표현 제거**:
- "~하는 것이다" → "~이다"
- "~한 바 있다" → "~했다"
- "~하는 것으로 나타났다" → "~로 나타났다"
- "~에 대해서" → "~에 대해" 또는 "~을"

**중복 제거**:
- "서로 간의 상호작용" → "상호작용"
- "미래에 대한 전망" → "전망"
- "과거의 역사" → "역사"

### 3. 일관성 (Consistency)
**원칙**: 문서 전체에서 용어, 표기, 스타일을 통일한다.

**용어 일관성**:
- 같은 개념은 같은 단어로 (예: "사용자" vs "유저" 통일)
- 전문 용어 첫 등장 시 설명, 이후 일관되게 사용
- 외래어 표기 통일 (예: "소프트웨어" vs "SW")

**문체 일관성**:
- 경어체(-습니다) 또는 평어체(-다) 통일
- 시제 일관성 (현재/과거)
- 문장 길이와 리듬의 균형

### 4. 정확성 (Accuracy)
**원칙**: 사실, 수치, 인용이 정확해야 한다.

**확인 사항**:
- 통계와 수치의 출처 명확
- 인명, 지명, 회사명 정확한 표기
- 날짜와 시간 정보 검증
- 인용문 원문과 대조

### 5. 독자 중심 (Reader-Centered)
**원칙**: 독자가 읽기 쉽고 이해하기 편하게.

**독자 배려**:
- 배경 지식 가정 최소화
- 전문 용어 사용 시 설명 추가
- 논리적 순서로 정보 배치
- 적절한 예시와 구체적 사례

## 문장 구조 개선 기법

### 주술 호응
**문제**: 주어와 술어가 맞지 않는 문장

**Before**: 이 보고서의 목적은 새로운 전략을 제안하는 데 있다.
**After**: 이 보고서는 새로운 전략을 제안한다.

### 능동태 우선
**원칙**: 피동형보다 능동형이 명확하고 힘있다.

**Before**: 이 문제는 팀에 의해 해결되었다.
**After**: 팀이 이 문제를 해결했다.

**Before**: 새로운 시스템이 도입되어졌다.
**After**: 새로운 시스템을 도입했다.

### 구체성
**원칙**: 추상적 표현보다 구체적 묘사

**Before**: 많은 사람들이 참여했다.
**After**: 500명이 참여했다.

**Before**: 빠른 시일 내에 완료할 예정이다.
**After**: 12월 15일까지 완료할 예정이다.

### 문장 길이 조절
**권장**: 한 문장 40-60자 (2줄 이내)
- 너무 긴 문장 → 두 문장으로 분리
- 너무 짧은 문장 연속 → 적절히 연결

**Before**: 이 프로젝트는 작년에 시작되었으며, 10명의 개발자가 참여했고, 6개월간 진행되었으며, 성공적으로 완료되었고, 많은 성과를 거두었다.

**After**: 이 프로젝트는 작년에 시작되어 6개월간 진행되었다. 10명의 개발자가 참여했으며, 성공적으로 완료하여 많은 성과를 거두었다.

## 논리 흐름 강화

### 단락 구조 (PEEL)
- **P**oint: 핵심 주장
- **E**vidence: 근거/예시
- **E**xplanation: 설명
- **L**ink: 다음 단락 연결

### 전환어 활용
**순서**: 첫째, 둘째, 마지막으로, 우선, 다음으로
**대조**: 그러나, 반면에, 하지만, 한편
**인과**: 따라서, 그러므로, 결과적으로, 왜냐하면
**추가**: 또한, 더불어, 게다가, 아울러
**강조**: 특히, 무엇보다, 중요한 것은

### 논리적 순서
1. **시간 순서**: 과거 → 현재 → 미래
2. **중요도 순서**: 중요한 것부터 또는 마지막에 강조
3. **일반-구체**: 전체 개요 → 세부 사항
4. **문제-해결**: 문제 제기 → 원인 분석 → 해결책 제시

## 설득력 있는 글쓰기

### 1. 구체적 증거 제시
- 통계와 수치
- 실제 사례와 예시
- 전문가 인용
- 연구 결과 참조

### 2. 논리적 근거
- 왜 그런지 설명
- 원인과 결과 명확히
- 가정과 전제 밝히기

### 3. 반론 예상과 대응
- 가능한 반론 인정
- 반론에 대한 답변
- 균형 잡힌 시각

### 4. 강력한 시작과 끝
**시작**: 독자의 관심을 끄는 질문, 통계, 일화
**끝**: 핵심 메시지 요약, 행동 촉구, 여운 남기기

## 단락 구성 원칙

### 단락 길이
- 일반 문서: 3-5문장 (150-200자)
- 웹 문서: 2-3문장 (짧게)
- 학술 문서: 5-7문장

### 단락 통일성
- 하나의 단락 = 하나의 아이디어
- 주제문 (Topic Sentence): 단락의 첫 문장에서 핵심 제시
- 뒷받침 문장: 주제문을 구체화, 예시, 설명
- 마무리: 단락 요약 또는 다음 단락 연결

### 단락 간 연결
- 논리적 흐름 유지
- 전환어로 자연스럽게 연결
- 반복되는 키워드로 연속성

## 문장 리듬과 호흡

### 문장 길이 변주
```
짧은 문장. 
약간 긴 문장으로 설명을 추가한다.
다시 짧게.
리듬감이 생긴다.
```

### 강조 기법
1. **반복**: 핵심 단어 반복으로 강조
2. **병렬**: 같은 구조 반복 (예: "~했고, ~했으며, ~했다")
3. **대조**: 상반된 개념 나란히 배치

### 읽기 쉬운 호흡
- 쉼표(,)로 짧은 호흡
- 마침표(.)로 긴 호흡
- 2-3문장마다 독자에게 숨 쉴 시간

## 비즈니스/기술 문서 특화

### 비즈니스 문서
**특징**: 명확성, 간결성, 행동 지향

**구조**:
1. 결론 먼저 (Bottom Line Up Front)
2. 핵심 메시지 강조
3. 구체적 숫자와 데이터
4. 다음 액션 명확히

**문체**:
- 전문적이면서 이해하기 쉽게
- 불필요한 수식어 제거
- 능동태와 현재형 선호

### 기술 문서
**특징**: 정확성, 체계성, 재현 가능성

**구조**:
1. 개요 (Overview)
2. 세부 내용 (Details)
3. 예시 (Examples)
4. 참고사항 (Notes)

**문체**:
- 명확한 용어 정의
- 단계별 설명
- 시각 자료 활용

## 편집 체크리스트

### 1차: 내용 검토
- [ ] 핵심 메시지가 명확한가?
- [ ] 논리적 흐름이 자연스러운가?
- [ ] 불필요한 내용이 있는가?
- [ ] 중요한 정보가 빠졌는가?

### 2차: 문장 검토
- [ ] 모든 문장이 명확한가?
- [ ] 주술 호응이 맞는가?
- [ ] 문장 길이가 적절한가?
- [ ] 능동태로 표현했는가?

### 3차: 표현 검토
- [ ] 용어가 일관적인가?
- [ ] 번역체 표현이 있는가?
- [ ] 불필요한 수식어가 있는가?
- [ ] 반복되는 표현이 있는가?

### 4차: 독자 관점 검토
- [ ] 독자가 이해하기 쉬운가?
- [ ] 전문 용어에 설명이 있는가?
- [ ] 예시가 적절한가?
- [ ] 읽기 편한 분량인가?

### 5차: 최종 검토
- [ ] 맞춤법과 띄어쓰기
- [ ] 문장부호
- [ ] 숫자와 날짜
- [ ] 고유명사
